## 1. 计算机系统概览 (System Overview)

### 1.1 什么是计算机体系结构？
刚开始学的时候，我容易把“组织”和“结构”搞混。现在的理解是：
*   **体系结构 (Architecture)**：程序员能看到的属性（比如 ISA 指令集、数据类型、寻址方式）。
*   **组织 (Organization)**：硬件如何实现这些属性（比如时钟频率、总线宽度、Cache 大小）。

**核心组件：**
1.  **CPU**：大脑（ALU 做运算，CU 做控制）。
2.  **存储器**：记忆（程序和数据都存在这里）。
3.  **I/O 子系统**：感官（键盘、屏幕、磁盘）。

> 💡 **我的思考：**  
> 为什么要有 Cache？课上说因为 CPU 太快，内存太慢。我查了一下数据：硬盘容量比寄存器大 4000 万倍，但速度却慢 2000 万倍。这种**速度 - 容量权衡 (Trade-off)** 贯穿了整个计算机设计。

### 1.2 存储层次与地址空间
笔记里提到“堆栈是存储设备但不用编址”，这点我当时有点疑惑。后来明白：**栈 (Stack)** 其实是在主存里划分的一块区域，它是**有地址**的，只是我们通过 **SP (Stack Pointer)** 寄存器来间接访问，不需要每次手动写地址。

**地址空间编址方式：**
*   **统一编址 (Memory Mapped I/O)**：I/O 设备像内存一样编址。优点是指令通用，缺点是占用内存地址空间。
*   **独立编址 (Isolated I/O)**：I/O 有独立的地址空间。需要专门的 `IN/OUT` 指令。

**RISC vs CISC 的存储访问：**
*   **RISC (如 ARM)**：只有 `LOAD/STORE` 指令能访问内存，运算只能在寄存器之间进行。
    *   *好处*：流水线设计简单，速度快。
    *   *代价*：代码密度可能稍低。
*   **CISC (如 x86)**：运算指令可以直接操作内存操作数。
    *   *好处*：汇编代码短。
    *   *代价*：硬件解码复杂。

### 1.3 总线 (Bus)
总线就是公共数据通路。关键点在于**仲裁 (Arbitration)**——每次只能有一个设备说话。
*   **带宽 (Bandwidth)**：单位时间传输量 (B/s)。
*   **延迟 (Latency)**：从请求到数据到达的时间。
*   **宽度**：一次能传多少位（32 位 vs 64 位）。

## 2. 数据表示与运算 (Data Representation)

这部分是基础，但最容易出错。我特意写了点代码验证一下。

### 2.1 二进制与补码
计算机怎么表示负数？**补码 (Two's Complement)** 是神来之笔。
*   **定义**：$P + Q = 2^n$。求补码方法：各位取反 + 1。
*   **为什么这么做？** 为了让减法变加法，简化硬件电路（不需要专门的减法器）。

**✅ 代码验证 (Python)：**
```python
def twos_complement(val, bits):
    """计算 val 的 bits 位补码表示"""
    if val < 0:
        val = (1 << bits) + val
    return f"{val:0{bits}b}"

# 验证 -3 的 8 位补码
print(f"-3 的 8 位补码：{twos_complement(-3, 8)}")  
# 输出：11111101 (符合课本：取反 00000010 -> 11111101)

# 验证加法：5 + (-3) = 2
a = 5
b = -3
# 模拟 8 位硬件加法（溢出自动截断）
res = (a + b) & 0xFF 
print(f"5 + (-3) 硬件结果：{res}") # 输出 2 ✅
```

**⚠️ 注意溢出：**
符号位相同，结果符号位不同 -> 溢出。比如两个正数相加变成负数。

### 2.2 浮点数 (IEEE 754)
这是我最晕的部分。为什么 `0.1 + 0.2 != 0.3`？
*   **结构**：`S (符号) | E (指数，偏置) | M (尾数，隐含 1)`
*   **偏置指数 (Biased Exponent)**：为了让指数比较大小更方便，不用处理负数符号位。单精度偏置值是 127。
*   **规格化**：尾数最高位总是 1，所以存储时可以省略（隐含位），省一位是一位。

**✅ 精度陷阱验证：**
```python
print(0.1 + 0.2 == 0.3)  # False! 😱
print(f"{0.1 + 0.2:.20f}") # 0.30000000000000004441
```
**原因：** 0.1 在二进制里是无限循环小数，存储时被截断了。

### 2.3 乘除法硬件实现
*   **乘法**：移位 + 加法。快速乘法可以用查找表，但太占空间。
*   **除法**：恢复余数法（Restoring Division）。如果减完是负数，商 0 并把除数加回去恢复。
*   **移位操作**：
    *   逻辑移位：补 0。
    *   算术移位：最高位补符号位（保持正负性）。

## 3. 指令集架构与 ARM (ISA & ARM)

这是本学期重点。ARM 的简洁设计让我对 RISC 有了实感。

### 3.1 指令格式与寻址
*   **操作码 (Opcode)**：做什么操作。
*   **操作数 (Operand)**：数据在哪。
*   **寻址方式**：
    *   **立即寻址**：`#100` (数据直接在指令里)。
    *   **直接寻址**：地址直接在指令里。
    *   **间接寻址**：寄存器里存的是地址（指针）。
    *   **基址 + 偏移**：`[Rn, #offset]` (数组访问常用)。

### 3.2 ARM 寄存器组织
ARM 有 16 个通用寄存器 (R0-R15)，但我发现它们并不是完全“通用”的：
*   **R13 (SP)**：栈指针 (Stack Pointer)。
*   **R14 (LR)**：链接寄存器 (Link Register)，存子程序返回地址。
*   **R15 (PC)**：程序计数器 (Program Counter)。
*   **CPSR**：状态寄存器 (存标志位 N, Z, C, V)。

> 💡 **踩坑记录：**  
> 一开始我以为 `LDR r0, #1000` 能直接加载大数，结果报错。后来才知道 ARM 指令只有 32 位，立即数有限制。大数要用伪指令 `LDR r0, =1000` 或者分次加载。

### 3.3 常用指令笔记
| 指令 | 含义 | 备注 |
| :--- | :--- | :--- |
| `LDR` | 加载 (Load) | 内存 -> 寄存器 |
| `STR` | 存储 (Store) | 寄存器 -> 内存 |
| `ADD` | 加法 | `ADD r0, r1, r2` (r0 = r1 + r2) |
| `CMP` | 比较 | 更新 CPSR 标志位，不存结果 |
| `B` | 跳转 | 无条件分支 |
| `BL` | 跳转并链接 | 子程序调用 (自动存返回地址到 LR) |
| `MOV` | 传送 | 寄存器间拷贝 |

**条件执行 (Condition Codes)：**
ARM 的特色！每条指令都可以加后缀，如 `ADDEQ` (相等才加)。
*   `EQ` (Z=1), `NE` (Z=0), `MI` (N=1), `PL` (N=0)...
*   **注意**：普通指令默认不更新标志位，要加 `S` 后缀 (如 `ADDS`) 才会更新 CPSR。但 `CMP` 例外，它隐含更新。

### 3.4 子程序调用
ARM 没有硬件堆栈自动管理调用，需要软件配合：
1.  **调用**：`BL label` (PC -> LR, 跳转到 label)。
2.  **返回**：`MOV PC, LR` (LR -> PC)。
3.  **嵌套调用**：因为 LR 会被覆盖，所以需要先把 LR 压栈 (`PUSH {LR}`)，返回前弹栈 (`POP {PC}`)。

**示例：一个简单的子程序框架**
```asm
MyFunc:
    PUSH {LR}       ; 保存返回地址
    ; ... 做事儿 ...
    POP {PC}        ; 恢复返回地址并跳转 (相当于 RET)
```

### 3.5 自动索引寻址 (Auto-indexing)
这个特性写数组遍历超方便：
*   **前索引**：`LDR r0, [r1, #4]!` (先 r1+4，再访问，**更新 r1**)
*   **后索引**：`LDR r0, [r1], #4` (先访问，再 r1+4，**更新 r1**)
*   **无更新**：`LDR r0, [r1, #4]` (不更新 r1)
> **心得**：那个感叹号 `!` 很容易忘，忘了就会导致指针不动，陷入死循环。

## 4. 控制单元与实现 (Control & Implementation)

### 4.1 存储程序概念 (Von Neumann)
*   指令和数据都存在内存里。
*   **PC** 指向下一条指令。
*   **执行周期**：取指 (Fetch) -> 译码 (Decode) -> 执行 (Execute)。
*   **访存次数**：至少 2 次（1 次取指令，1 次取/写数据）。

### 4.2 RTL (寄存器传输语言)
用来描述硬件操作的语言。
*   `[15]` 表示地址 15 的内容。
*   `<-` 表示数据传送。
*   例：`[PC] <- [PC] + 4` (ARM 指令长 4 字节)。

### 4.3 乱序执行 (Out-of-Order)
高级 CPU 不用傻等第一条指令做完。只要数据不冲突，后面的指令可以先执行。这是提升 IPC (Instructions Per Cycle) 的关键。
